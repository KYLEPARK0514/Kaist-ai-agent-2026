□ 사업개요
EduPath AI는 학생의 학사 의사결정을 지원하는 서비스가 아니라, 실질적으로 학습·수강·진로 선택의 판단 과정을 구조화하고 자동화하는 Agent를 구축하는 사업이다.
현재 대학생과 대학원생은 다음과 같은 구조적 비효율을 겪고 있다.
첫째, 실러버스와 졸업요건, 선수과목, 트랙 규정, 행정 규정이 각기 다른 문서와 시스템에 분산되어 있어 전체 맥락을 파악하기 어렵다.
둘째, 과목 선택은 종종 선배 추천이나 체감 난이도 위주로 결정되며, 실제 커리어 목표와의 연결은 매우 약하다.
셋째, 학교의 기존 시스템은 공지·수강신청·성적조회 등 거래 기능 위주이기 때문에 학생의 “의사결정” 자체를 도와주지 못한다.
넷째, 학과 행정실은 반복적인 문의에 응답하지만, 이는 고부가가치 행정이 아니라 정보 전달의 반복에 가깝다.
EduPath AI는 이러한 문제를 해결하기 위해
“학사 데이터 수집 → 구조화 → 개인 이수현황 반영 → 규정 검증 → 과목 분석 → 추천 → 리스크 경고 → 학습 경로 설계”
라는 흐름을 하나의 시스템 안에서 제공한다.
즉 EduPath AI는 단순 챗봇이나 검색 엔진이 아니라, 학생에게는 학사 비서이자 진로 설계사, 대학에게는 행정 효율화와 교육 품질 고도화를 위한 인프라로 기능하는 플랫폼이다.

1. 문제 인식
1.1. 창업 아이템의 필요성
창업자는 실제 학업 과정에서 학생이 겪는 핵심 어려움이 정보의 부재가 아니라, 정보의 분산·비정형성·해석 부담·연결 부재라는 점에 주목하였다. 학생은 실러버스를 하나하나 내려받아 읽고, 졸업요건을 별도 규정집에서 확인하며, 최종적으로는 행정실이나 선배에게 물어보는 과정을 반복한다. 이 과정은 시간이 많이 들 뿐 아니라, 학생마다 정보 접근성과 해석 능력 차이가 커 결과적으로 학업 성과와 진로 설계에 불균형을 낳는다.
특히 대학원이나 특수대학원, 복수 트랙 운영 학과에서는 과목 수 자체보다 “과목 간 관계”와 “졸업요건 충족 로직”이 더 복잡해, 단순 조회 기능만으로는 의사결정이 불가능하다. 학생은 다음과 같은 문제를 동시에 겪는다.
구분	현재 시장의 문제	해결 방향
정보 접근	실러버스, 규정, 공지가 분산되어 있음	데이터 통합 및 단일 인터페이스 제공
의사결정	무엇을 들어야 하는지 스스로 판단해야 함	AI 기반 수강 설계 및 검증
졸업 관리	요건 누락 위험이 존재함	Rule Engine 기반 자동 검증
커리어 연계	과목과 직무 간 연결이 약함	스킬-과목 매핑 기반 추천
학습 구조	선수과목 외 실제 선후 관계 파악이 어려움	실러버스 기반 학습 경로 생성
행정 부담	반복 문의가 많음	FAQ 수준을 넘는 Agent 자동응답
따라서 본 아이템의 필요성은 단순 편의성 개선이 아니라, 학생의 잘못된 선택 비용을 줄이고 대학의 반복 행정 비용을 구조적으로 절감하는 데 있다.

1.2. 시장규모 및 성장성
EduPath AI는 전통적인 LMS 시장이 아니라, 학사 정보·개인화 추천·교육 AI가 만나는 교차 영역에 위치한다. 국내외 대학은 이미 수강신청, LMS, 학생포털 등의 기초 시스템은 갖추고 있으나, 이를 바탕으로 학생의 개별 상황에 맞는 학습 경로와 진로 정렬을 제공하는 계층은 여전히 미비하다. 특히 생성형 AI의 확산으로 단순 정보 검색을 넘어, 구조적 판단과 개인화 추천에 대한 기대 수준이 높아지고 있다.
본 서비스는 대학과 대학원뿐 아니라 향후 MOOC, 사내교육, 자격과정, 부트캠프 등으로 확장 가능한 범용성을 가진다. 초기 시장은 대학/대학원 내 학사 의사결정 수요가 집중된 조직 단위로 설정하되, 장기적으로는 교육기관 전반의 학습 설계 플랫폼으로 성장 가능하다.

2.2. 핵심 기능
EduPath AI의 MVP는 “정보는 가장 적게 찾고, 의사결정은 가장 정확하게”를 목표로 한다. 모든 기능을 동시에 완성도 높게 구현하기보다, 학생과 대학이 가장 즉각적으로 가치를 체감할 수 있는 핵심 기능을 우선순위 기반으로 설계한다. 아래 기능은 서로 독립된 모듈이 아니라, 실제로는 하나의 학사 의사결정 흐름을 구성하는 계층이다.
1) 수강 설계 자동화
이 기능은 EduPath AI의 중심 기능이다. 학생이 현재까지 이수한 과목, 남은 학기 수, 선호 학점 부하, 목표 분야를 입력하면, 시스템은 졸업요건과 선수과목 조건을 반영하여 학기별 최적 수강 계획을 제안한다. 단순히 남은 과목을 나열하는 것이 아니라, 필수과목 우선순위, 선택과목의 커리어 적합성, 과목 간 난이도 균형, 특정 학기 과부하 여부까지 종합적으로 고려한다.
세부적으로는
•	이미 이수한 과목과 중복되는 선택을 제거하고 
•	필수 누락 과목을 우선 반영하며 
•	특정 학기별 학점 상한과 workload를 균형 있게 배치하고 
•	학생의 관심 분야(예: AI, 금융, 전략, 데이터)에 따라 선택과목 가중치를 다르게 계산한다. 
이를 통해 학생은 “무엇을 들을지” 뿐 아니라 “언제 들어야 하는지”까지 한 번에 확인할 수 있으며, 대학 입장에서는 수강설계 관련 반복 상담을 줄일 수 있다.
2) 실러버스 기반 과목 분석 (RAG)
기존 학생은 과목명을 보고 추상적으로 판단하거나, 실러버스 PDF를 열어 직접 읽어야 했다. 본 기능은 실러버스를 Vector DB에 적재하고, 과목 설명·주차별 내용·평가 방식·과제 구조를 LLM과 결합하여 질문 가능한 형태로 바꾼다.
학생은 다음과 같은 질문을 할 수 있다.
•	“A 과목과 B 과목은 실제로 어떤 차이가 있는가” 
•	“이 과목은 수학적 배경이 많이 필요한가” 
•	“발표/팀플 비중이 높은가” 
•	“실무와 연계성이 높은가” 
시스템은 단순 요약이 아니라, 실러버스 문맥에 기반해 강의 내용, 난이도, workload, 평가 구조, 학습 성격을 설명한다. 또한 두 과목 이상을 비교하여 어떤 학생에게 더 적합한지까지 제시할 수 있다. 이는 학생의 정보 탐색 비용을 급격히 줄여주며, 특히 선택과목이 많은 과정에서 체감 가치가 크다.
3) 졸업요건 자동 검증 (Rule Engine)
이 기능은 정확성과 신뢰성이 가장 중요한 영역이다. 각 학과·과정의 졸업규정, 필수·선택 구분, 트랙 요건, 최소 학점, 중복 인정 여부 등을 정형화된 규칙으로 관리하고, 학생의 현재 이수현황과 매칭하여 졸업 가능 여부를 자동 검증한다.
단순히 “가능/불가”만 보여주는 것이 아니라,
•	현재 충족한 요건 
•	아직 부족한 필수 과목 
•	남은 선택학점 
•	특정 트랙 요건 충족 여부 
•	중복 계산 불가 항목
을 구체적으로 제시한다. 
이 기능의 핵심 가치는 학생의 졸업 리스크를 사전에 줄이는 데 있다. 동시에 행정실이 반복적으로 응답해야 하는 “제가 졸업 가능한가요?”, “이 과목이 트랙 인정되나요?”와 같은 질문을 자동화할 수 있어 대학 측 효용도 높다.
4) 커리어 연계 추천
대부분의 학사 시스템은 과목을 학점 단위로만 관리하고, 해당 과목이 어떤 직무 역량과 연결되는지 보여주지 않는다. EduPath AI는 과목별 학습 내용과 실무 역량을 스킬 단위로 매핑하여, 학생의 목표 직무에 맞는 과목을 추천한다.
예를 들어
•	컨설팅 지향 학생에게는 문제해결, 프레젠테이션, 데이터 해석 관련 과목을 
•	AI 직무 지향 학생에게는 통계, 머신러닝, Python, 데이터엔지니어링 관련 과목을 
•	금융권 지향 학생에게는 재무, 리스크, 데이터 분석, 규제 관련 과목을
상대적으로 높은 우선순위로 제시한다. 
또한 추천 이유를 설명 가능한 방식으로 제공한다. 단순 추천이 아니라 “이 과목이 어떤 스킬을 길러주며, 왜 해당 직무와 연결되는지”를 보여줘야 학생의 수용성이 높아진다. 이 기능은 장기적으로 취업센터, 커리어센터, 비교과 프로그램과의 연계 가능성도 높다.
5) 리스크 경고 시스템
많은 학생이 수강신청 단계에서 장기 리스크를 체감하지 못한다. 예를 들어 필수 과목을 뒤로 미루다가 졸업 직전에 몰리는 경우, 선수과목 없이 상위 과목을 신청하는 경우, 한 학기에 고난도 과목만 몰아 넣는 경우가 빈번하다. 본 기능은 추천 이전보다 오히려 “방지” 측면에서 큰 가치를 가진다.
시스템은
•	선수과목 미이수 상태에서 상위 과목 수강 시도 
•	특정 학기 workload 과다 편성 
•	필수요건 지연으로 인한 졸업 리스크 
•	과목 조합 상 학습 성과 저하 가능성
을 사전에 감지하고 경고한다. 
중요한 점은 단순 경고창이 아니라, 왜 위험한지와 대안 조합을 함께 제시하는 것이다. 예를 들어 “이번 학기에는 A, B, C를 동시에 듣기보다 A+B 후 다음 학기에 C를 듣는 편이 학습 부담과 선수지식 측면에서 더 적절함”과 같이 실질적 행동 대안을 제공해야 한다.
6) 실러버스 기반 학습 선후 관계 분석
이 기능은 EduPath AI의 핵심 차별화 요소다. 공식 선수과목 체계는 행정적 최소조건일 뿐, 실제 학습 이해 순서와 일치하지 않는 경우가 많다. 본 기능은 실러버스에서 핵심 개념과 스킬을 추출하고, 과목 간 지식 의존 관계를 분석하여 “공식 선수과목”을 넘어선 실제 학습 선후 관계를 제시한다.
예를 들어 특정 과목이 공식 선수과목은 없지만 통계 기초와 Python 이해가 선행되어야 실질적으로 따라갈 수 있다면, 시스템은 이를 바탕으로 추천 순서를 설계한다. 반대로 행정상 선수과목이 있더라도 실제 내용 중복이 크거나 우회 가능한 경우에는 유연한 학습 경로를 제안할 수 있다.
이 기능의 가치는 세 가지다.
첫째, 학생이 과목 간 연결 구조를 이해하고 장기적 학습 전략을 세울 수 있다.
둘째, 학과는 커리큘럼의 중복·단절·비효율을 데이터 기반으로 점검할 수 있다.
셋째, 서비스가 단순 수강 추천 툴이 아니라, 실질적인 “학습 설계 엔진”으로 포지셔닝될 수 있다.

2.3. 기술 구현 방법
EduPath AI의 기술 구현은 크게 5개 영역으로 구성된다. 핵심은 “질문에 답하는 챗봇”이 아니라, 정형 규칙과 비정형 의미를 함께 해석하는 복합 의사결정 구조를 만드는 것이다.
기술 영역	활용 기술	역할
데이터 수집/정제	크롤링, PDF 파싱, 문서 전처리	실러버스/규정/과목정보 수집 및 구조화
Vector & RAG	Embedding, Vector DB	실러버스 검색 및 문맥 기반 질의응답
Rule Engine	커스텀 로직/정책 테이블	졸업요건, 트랙, 필수/선택 규정 검증
Graph Engine	개념/스킬 추출, 관계 분석	과목 간 선후관계 및 학습 경로 생성
LLM 응답 계층	GPT 계열 모델	추천 결과 설명, 과목 비교, 사용자 친화 응답 생성
또한 본 서비스에서 중요한 것은 단순 정확도보다 데이터 표준화다. 학과별로 유사한 의미의 과목군이 서로 다른 표현을 사용하고, 실러버스 형식도 상이하므로, 이를 하나의 구조로 통합하는 내부 스키마 설계가 필수적이다. 예를 들어 과목별 속성을 “핵심개념/선행개념/평가방식/워크로드/실무연계/스킬태그”와 같은 공통 구조로 매핑해야 추천과 비교, 관계분석이 가능해진다.
또한 LLM 응답은 100% 정확성을 전제로 하지 않으므로, 추천 결과에는 반드시
•	근거 과목/규정 
•	충족/미충족 사유 
•	불확실 시 사용자 확인 필요 항목
을 함께 제공하도록 설계할 계획이다. 장기적으로는 관리자 검수 UI를 통해 규칙과 매핑 구조를 지속적으로 보정하는 체계를 구축할 수 있다. 
[기술적 경쟁력]
● 실러버스, 규정, 커리어 데이터를 하나의 의사결정 구조로 통합
● Rule Engine과 LLM을 결합하여 정확성과 유연성을 동시에 확보
● 과목별 스킬 태깅 및 개념 추출을 통한 설명 가능한 추천 제공
● 실러버스 간 dependency graph 생성으로 실제 학습 순서 설계 가능
● 단순 챗봇이 아닌 학사 의사결정 인프라로 확장 가능한 구조 확보
